# 2026-07-24 — Geolith → WebAssembly (contrato wasm-gaming)

Port de **Geolith** (emulador Neo Geo AES/MVS de Rupert Carmichael,
https://gitlab.com/jgemu/geolith) a WebAssembly, empaquetado como engine SDK
conforme a [`@wasm-gaming/engine-specs`](https://github.com/wasm-gaming/engine-specs),
igual que fbneo-wasm y hermanos.

**Estado: FUNCIONAL de punta a punta.** Metal Slug (`mslug.neo` + `neogeo.zip`)
arranca, renderiza pixel-perfect, corre a ~59.19fps (audio-clocked) con sonido,
input de teclado, savestates y persistencia OPFS. Verificado headless (Node) y
en navegador real (Playwright/Chromium), incluida la demo shell completa.

---

## 1. Arquitectura (distinta a FBNeo — mucho más limpia)

Geolith es un core **C11 puro sin frontend** (su único binding es jg.c, del
Jolly Good API). En vez de compilar el frontend SDL con ASYNCIFY, aquí:

- **`shim/geo_shim.c`** reimplementa el cableado de jg.c contra los headers del
  core (`geo.h`, `geo_lspc.h`, `geo_mixer.h`, `geo_neo.h`) con ABI plano:
  `geowasm_setup/load_bios/load_rom/reset/exec/frame_rgba/input/...`
- **Sin SDL, sin ASYNCIFY, sin GL emulado, sin FS** (`-sFILESYSTEM=0`).
  El SDK JS llama `_geowasm_exec()` por frame, blitea RGBA a canvas 2D y
  encola audio int16 a un `AudioWorklet` (ring buffer).
- **Pacing por audio**: se ejecutan frames solo cuando el buffer de audio baja
  de ~90ms → velocidad clavada sin drift. Fallback a wall-clock si el
  AudioContext está suspendido (autoplay).
- Resultado: `geolith.wasm` = **418KB** (vs 38MB de fbneo), build en ~1 min,
  ~336fps headless en una M-series (5.7× realtime).

### Datos del core que definen el diseño
- Vídeo: buffer XRGB8888 320×264 (pitch 320), visible 304×224 en (8,24) con
  máscara de overscan 8px (`geo_lspc_set_buffer`).
- Audio: mixer YM2610 → resampler Speex → int16 estéreo. **La constante
  SAMPLERATE_YM2610=56319 asume 60Hz**: hay que pedir
  `sampleRate * 60/framerate` a `geo_mixer_set_rate()` para obtener
  exactamente `sampleRate/framerate` frames por exec (es el
  SAMPLERATE_ADJUSTED=44396 de jg.c). Sin esto va un ~1.4% lento.
- ROM: `geo_neo_load()` **aliasa** los datos (no copia) → el buffer .neo debe
  vivir toda la sesión. `geo_bios_load_mem()` acepta el **zip** MAME directo
  (miniz vendored) y extrae a heap propio → se puede liberar el staging.
- Orden de boot (de jg.c): set system/region → mixer rate/init → geo_init →
  bios → neo → restore savedata → `geo_reset(1)` (el 68K lee vectores de ROM).
- Inputs por callbacks: activo-bajo por puerto (js/stat_a/stat_b/systype/dipsw);
  el shim mantiene máscaras activo-alto seteadas desde JS.
- Savedata sin FS: `geo_mem_ptr(NVRAM=4/CARTRAM=5/MEMCARD=7)` → el SDK persiste
  esos bytes a OPFS (`geolith/<namespace>/`).
- BIOS por región/sistema (geo.c): MVS-US=`sp-u2.sp1`, EU=`sp-s2.sp1`,
  UNI=`uni-bios_4_0.rom`, AES=`neo-epo.bin`; siempre `000-lo.lo` + (MVS/UNI)
  `sfix.sfix`+`sm1.sm1`.

## 2. Formato .neo y conversor

Geolith solo carga **TerraOnion NeoSD .neo** (header 4096B: "NEO\x01", tamaños
P/S/M/V1/V2/C LE, nombre/NGH; luego regiones concatenadas). Para testear se
escribió `scripts/mame2neo.py` (mslug.zip → mslug.neo):
- P: address-ordered; un p1 de 2MB (ROM_CONTINUE, ej. mslug) va con **mitades
  intercambiadas**.
- C: pares (c1,c2) **byte-interleaved** (como ROM_LOAD16_BYTE de MAME) —
  confirmado en geo_lspc.c ("Interleaved odd/even byte order [0,2,1,3]").
- V: v-roms concatenadas en V1, v2sz=0 (región ADPCM compartida).
- OJO: no es un NeoBuilder completo (sin DB de parches por juego ni cifrados
  SMA/PCM2) — para sets complejos usar NeoBuilder real.

## 3. Build

- `scripts/build-geolith.sh`: clona ref pinneada (`4c0db390`, v0.4.2), compila
  la lista CSRCS del Makefile upstream + deps vendored (miniz, speex
  resample.c) + shim, directo con emcc. **Cero parches al upstream.**
- Flags clave: `-sMODULARIZE -sEXPORT_NAME=createGeolithModule`
  `-sENVIRONMENT=web,node` (node para el smoke test)
  `-sALLOW_MEMORY_GROWTH -sINITIAL_MEMORY=128MB` (carts hasta ~96MB)
  `-sFILESYSTEM=0 -sEXPORTED_RUNTIME_METHODS=HEAPU8,HEAP16,HEAPU32`.
- `make build-wasm` (Docker emscripten/emsdk), `build-sdk`, `preview` (:8028).

## 4. Tests (repetibles)

- `node scripts/smoke-test.cjs` — headless: boot MVS/US + mslug, 900 frames,
  valida samples/frame (1622 @48kHz), colores del framebuffer, velocidad.
  (Nota: requiere `"type":"module"` workaround — copia geolith.js a .cjs.)
- Browser (scratchpad, playwright-core + Chromium cacheado de ms-playwright):
  test.html importa el SDK, fetch de rom/bios, `--autoplay-policy=
  no-user-gesture-required`; verifica ~59fps sostenidos, savestate roundtrip
  (222.757 bytes), coin(5)+start(1) por teclado llegan al juego (screenshot
  in-game), y la demo shell completa (setInputFiles + Launch).

## 5. Pendiente / mejoras

- **Gamepad**: solo teclado; falta mapear la Gamepad API.
- **CD/CDZ**: fuentes compiladas pero sin API de disco expuesta (cue/chd, BIOS
  neocd/neocdz, mixer CDDA).
- **Inputs especiales**: mahjong, trackball (Irritating Maze), V-Liner, 4P
  FTC1B — el shim solo cablea joysticks estándar.
- **`irrmaze.zip` aux BIOS** (geo_bios_load_mem_aux) no expuesto.
- **Opciones no expuestas**: paleta raw, overclock (div68k), adpcm_wrap off,
  watchdog tolerance, unihw (AES vs MVS bajo UniBIOS).
- **CI**: faltan workflows (.github) espejo de los hermanos, y commit inicial
  (el repo no tiene git aún).
- El `?bios=required` de la demo asume neogeo.zip; con `system:'aes'` el
  usuario debe subir aes.zip (documentado en README).

## Archivos clave
- `shim/geo_shim.c` — todo el puente C (vídeo RGBA, mixer rate 60Hz-adjust,
  inputs, savestates, savedata).
- `src/geolith.sdk.ts` — SDK contrato (AudioWorklet, pacing, OPFS, teclado).
- `scripts/build-geolith.sh` — build emcc directo, ref pinneada.
- `scripts/mame2neo.py` — conversor de test MAME→.neo.
- `romset/` — mslug.neo + neogeo.zip para tests locales (no publicar).
